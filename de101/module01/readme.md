## Шаг объединение данных в одну таблицу

Объединять данные будем во вкладке Orders. Там по умолчанию больше всего данных, 
есть общие поля с другими таблицами: "Region", "Order ID", "Product ID". Ну и самое важное 
именно здесь у нас минимальная грануляция. Значит, мы сможем построить более 
детализированные отчеты.

1 Для начала добавим недостающие столбцы "Person" и "Returned"

Чтобы подтянуть данные из вкладки People, можно воспользоваться функцией "Если"
в excel. Формула будет выглядеть так:

`=ЕСЛИ([@Region]="West"; "Anna Andreadi"; ЕСЛИ(M3="East"; "Chuck Magee"; ЕСЛИ(M3="Central"; "Kelly Williams"; ЕСЛИ(M3="South"; "Cassandra Brandow"; "not_date!!!"))))`

тут:

- [@Region]="West" - это обращение к полу таблицы на текущей странице
- M3 - это обращение к ячейке М2 на текущей странице

В итоге получаем таблицу, где для каждого региона указан его менеджер.

Чтобы подтянуть данные по возвратам из вкладки Returns, можно 
воспользоваться сразу несколькими функциями ИНДЕКС, ПОИСКПОЗ, ЕСЛИОШИБКА, ВПР,
ПРОСМОТРХ (XLOOKUP)

В качестве альтернативы можно использовать функцию ВПР (VKLOOKUP).
Мой вариант формулы такой:

`=ПРОСМОТРX(B2;t_returned[Order ID]; t_returned[Returned];"No")`

- первый аргумент - что искать
- второй аргумент - где искать
- третий аргумент - что вывести при нахождении совпадения
- четвертый [необязательный] - что вывести в случае ненахождения совпадений
